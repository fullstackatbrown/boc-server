GENERAL NOTES:
Responses will never include userIds, but tripIds are exposed (client needs them to request trip specific data).
Trip status progresses as follows: Staging -> Open -> Pre-Trip -> Post-Trip -> Complete. Once a trip has changed status, it cannot revert back.

EXPRESS ROUTES:                         METHOD: AUTH REQUIRED:  BODY:           RESPONSE BEHAVIOR:
/ OR <undefined_route>                  GET     None            N/A             Returns a 404 with a brief welcome message explaining that the route
                                                                                was not found
/trips                                  GET     None            N/A             Sends back all trip data (except planningChecklist) for all non-Staging
                                                                                trips whose plannedDate falls within the current semester (Jan-May or
                                                                                Sep-Dec). Returns an empty array during summer months (Jun-Aug).
/leaders                                GET     None            N/A             Sends back a list of all the names and emails of users who are
                                                                                leaders/admin
/user/                                  GET     Any Role        N/A             Sends back basic user data (firstName, lastName, email, phone, role,
                                                                                hasWaiver, tripsLead, tripsParticipated, joinedListserv) for the
                                                                                authenticated user. Does not include TripSignUps.
/user/profile                           GET     Any Role        N/A             Sends back all user data (except lotteryWeight) for user
                                                                                and all signup data for all associated trip signups
/user/add-phone                         POST    Any Role        Phone Json      Stores phone number for user, overriding any previous entry and deleting
                                                                                all non-numeric characters before inserting
/user/listserv-add                      POST    Any Role        N/A             Appends user's email to listserv-additions.txt and sets joinedListserv=true.
                                                                                No-op if user has already joined the listserv.
/trip/<tripId>                          GET     None*           N/A             If user not logged in or not signed up for trip: sends back all trip
                                                                                data for tripId (except planningChecklist) as long as the trip status
                                                                                is not Staging (returns 401 if Staging). If user is signed up with
                                                                                tripRole Participant, sends back same trip data plus all associated
                                                                                signup data. If user is signed up with tripRole Leader, executes same
                                                                                behavior but includes planningChecklist, a list of all leaders' names
                                                                                and emails, and a list of Selected/Attended/No Show participants'
                                                                                signups (if trip status is Pre-Trip or later).
/trip/<tripId>/is-signed-up             GET     Any Role        N/A             Sends back a boolean expressing whether or not a user has a signup
                                                                                instance for the trip with the specified id. Will return true for
                                                                                leaders leading the trip and even for participants who signed up but
                                                                                weren't selected (as long as they still have a signup instance).
/leader/create-trip                     POST    Leader+         Creation Json   Creates a private (Staging) trip using the data in the request body
                                                                                and returns the created trip JSON (status 200).
/trip/<tripId>/lead/task                POST    Trip Leader     Task Json       Modifies planningChecklist for trip at <tripId> accordingly. If all tasks
                                                                                are marked complete, sets trip status to Complete. Note: "Lottery" and
                                                                                "Attendance" are automated tasks whose complete field cannot be manually
                                                                                toggled (they are marked complete automatically by the server).
/trip/<tripId>/lead/alter               POST    Trip Leader     Alter Json      Modifies the data associated with the trip at <tripId> accordingly. Only
                                                                                plannedDate may be editted once trip is in Pre-Trip state (and nothing may
                                                                                be editted after that). class/priceOverride may not be editted once Open
                                                                                status is reached.
/trip/<tripId>/lead/open                POST    Trip Leader     N/A             Sets trip status to Open, assuming its current status is Staging. May not
                                                                                be performed if blurb or sentenceDesc is still null (not completed).
/trip/<tripId>/lead/lottery             POST    Trip Leader     N/A             Selects up to the max_size of the trip at <tripId> participants who have
                                                                                signed up for the trip. Updates the status of all signed up participants
                                                                                accordingly, updating lottery weights as well. Sets status to Pre-Trip
                                                                                for the trip. Sends back a list of emails of accepted participants, one
                                                                                of waitlisted participants, and a list of emails of rejected participants.
                                                                                Currently, ALL non-selected participants are placed on the waitlist
                                                                                (the notAccepted list will always be empty). Selected participants'
                                                                                elevated lotteryWeights are reset to 1; non-selected participants
                                                                                receive a lotteryWeight bonus. May only be performed if trip status
                                                                                is currently Open.
/trip/<tripId>/lead/participants        GET     Trip Leader     N/A             Returns a list of all Participant-role signups on a trip regardless of
                                                                                signup status. Each entry includes firstName, lastName, email, status,
                                                                                confirmed, and paid.
/trip/<tripId>/lead/add-participant     POST    Trip Leader     N/A             Adds a participant from the waitlist to the list of selected participants,
                                                                                prioritizing confirmed participants. Will return a JSON object with a single
                                                                                field "success" which will be set to 0 if there are no participants on the
                                                                                waitlist left and 1 otherwise. May only be performed if trip status is
                                                                                currently Pre-Trip.
/trip/<tripId>/lead/remove-participant  POST    Trip Leader     Remove Json     Removes the participant specified by email from the trip (takes their Selected
                                                                                status and turns it into Not Selected). Can only be performed if the trip
                                                                                status is currently Pre-Trip and the specified email matches a selected
                                                                                participant.
/trip/<tripId>/lead/attendance          POST    Trip Leader     Attendance Json Updates status of attendant participants and noshows, updating lottery
                                                                                weights accordingly. Deletes signup instances of excused absents.
                                                                                Increments tripsLead for all trip leaders. Sets trip status to Complete.
                                                                                May only be performed if trip status is currently Post-Trip and current
                                                                                datetime is on or after the trip's plannedDate.
/trip/<tripId>/signup                   POST    Any Role        N/A             Creates a new TripSignUp instance for the user on the trip at <tripId>.
                                                                                May only be performed if the trip has Open status.
/trip/<tripId>/participate/confirm      POST    Any Role        N/A             Sets confirmed=true on the user's TripSignUp for the trip at tripId.
                                                                                Indicates willingness/readiness to attend. Assumes any required payment
                                                                                has been made (although there is no way to enforce this on this server).
/trip/<tripId>/participate/cancel       POST    Any Role        N/A             Destroys the user's TripSignUp instance for the trip at tripId.
/trip/<tripId>/participate/pay          POST    Any Role        N/A             Sets paid=true on the user's TripSignUp for the trip at tripId.
/public/leader-stats/:first/:last       GET     None            N/A             Returns { totalTrips: <count> } representing the total number of trips
                                                                                the specified leader has led. (:first and :last are firstName/lastName)
/public/leader-trips/:first/:last       GET     None            N/A             Returns an array of trips led by the specified leader. Each entry
                                                                                includes tripId, tripName, date, sentenceDesc, and lotteryInfo
                                                                                (hardcoded to "Hosted Trip"). (:first and :last are firstName/lastName)
/admin/alter-user                       POST    Admin           Role Json       Updates role of target user accordingly. If a leader/admin is demoted, all
                                                                                trip signups in which they had the role of trip leader are deleted.

FORM/JSON DESCRIPTIONS:
Phone Json:
{
    phoneNum: <phone_num> //No need to sanitize - all non-numeric characters are filtered out automatically
}

Creation Json:
{
    leaders: [<leader_email1>, ...], //Not necessarily including the email of the creating leader, whose role is implicit
    tripName: <trip_name>,
    category: <category>, //One of: Hiking, Camping, Backpacking, Biking, Climbing, Skiing, Water, Event, Running, Exploration, Local, Special
    plannedDate: <date_obj>, //Specific up to the day of the trip (but not necessarily the hour)
    plannedEndDate: <date_obj>, //Can be null; use for multi-day trips
    maxSize: <max_num_participants>,
    class: <price_class>, //Options include characters A-J (see marketplace) plus Z (for free trips)
    priceOverride: <price_float>, //Only class or priceOverride should be defined, not both (but exactly one must be)
    sentenceDesc: <sentence_long_description>, //Can be null
    blurb: <paragraph_long_description>, //Can be null
    image: <image_url>, //Can be null
}

Task Json:
{
    task: <name_of_task>,
    responsible: <email_of_leader>, //May NOT be null - use empty string if no one is responsible yet
    complete: <boolean>,
}

Remove Json:
{
    email: <participant_email>,
}

Alter Json:
{
    <any_trip_creation_field>, //Except leaders (all fields from Creation Json minus leaders are valid)
    newLeader: <leader_email>, //Optional; adds one new leader to the trip (different from leaders[] in creation)
    //Multiple fields may be included, but none are required
}

Attendance Json:
{
    selectedParticipants: {
        <participant_email>: <attendance_status>, //attendance_status can be any of 'Attended', 'Excused Absence', or 'No Show'
        ...
    },
    additionalParticipants: [
        <participant_email>, //Participants who attended but were not on the selected list; signups are created for them
        ...
    ]
}

Role Json:
{
    role: <web_role>, //May be any of Participant, Leader, or Admin
    email: <email_of_user_to_alter>,
}
